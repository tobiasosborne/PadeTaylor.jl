## Design synthesis

Fig 3 reuses Fig 2 column 3's A4 cross-mode + A5 sheet-aware Stage-2
solve for the SAME PVI problem at (`α,β,γ,δ`) = (4, -4, 8, -8),
`u(10) = 0.429534…`.  Differences vs Fig 2:

  - **Wider ζ-window**: `Re ζ ∈ [-1.5, 2.5]` vs Fig 2's `[0.05, 3.0]`
    (still narrower than FFW's full `[-4.6, 2.3]` from md:183-189; v1
    corner — see "What is NOT shipped").
  - **Phase portrait** rendering: `arg w(ζ)` and `arg u(z)` plotted
    with HSV colormap in `[-π, π]`, same pattern as Fig 4's third
    column (worklog 038).
  - **z-plane Cartesian-resample**: per-sheet phase portraits in the
    z-plane via `ζ = log|z| + i (arg z + 2π·s)` bilinear interp from
    the rendered `W` array, lifted from Fig 4.

### rng_seed fragility (honest v1 corner)

The walker's `shuffle(rng, targets)` (FW 2011 line 156) means the
target traversal order — and thus the chance of the winding ring
producing cut-crossings — depends on `rng_seed`.  A small sweep
during development showed:

  | seed | visited | sheet counts |
  |------|---------|--------------|
  | 0    | 247     | `{[0]=>247}`  ←  package default, NO crossings |
  | 1    | 272     | `{[0]=>248, [-1]=>24}` |
  | 2    | 526     | `{[0]=>477, [1]=>49}`  ←  used |
  | 17   | 266     | `{[0]=>225, [1]=>41}` |
  | 42   | 1435    | `{[0]=>1408, [1]=>27}` |

Fig 3 ships with `rng_seed = 2`.  This is the same kind of rng
fragility flagged in worklog 044; the principled fix (Poisson-disk
node placement that naturally circumambulates the branch point) is
the same deferred infrastructure as bead `padetaylor-zwh`.

### Test-time tolerance vs figure-time tolerance

The figure script uses the full Fig 2 / Fig 3 winding ring (8
targets) + dense base (Re-step 0.35, Im-step 0.7).  The test reuses
the EXACT same target set + seed so that the captured reference
values (FF3.1.3, FF3.1.6) pin the figure-script-equivalent run.
Test wall ≈ 11 s — slightly more than Fig 2's test (~2 s) because
the wider Im-window + winding-ring forces more walker work; still
well under the worklog-038-style 30-s budget for figure tests.

## Mutation-proof results

Three load-bearing mutations applied + restored per CLAUDE.md Rule 4.

  | Mutation | Bite count (of 20 new assertions)    | Where it hits |
  |----------|---------------------------------------|---------------|
  | M1       | 15 ERROR + 5 GREEN remaining          | Walker corners under α=5: "target unreachable in 4000 steps"; FF3.1.2 onward all error |
  | M3       | 15 ERROR + 5 GREEN remaining          | Walker corners on cut: "all 5 wedge candidates failed" |
  | M4       | 5 RED + 1 ERROR + 14 GREEN remaining  | FF3.1.2 (sheet [1] NaN), FF3.1.5 (no non-[0] sheets), FF3.1.6 (3 RED on Δarg = NaN) |

M1 is STRONGER than the plan predicted — the α-corruption doesn't
just shift the captured reference numbers (which would be the
canonical FF3.1.3 absolute-pin bite); it destabilises the walker
itself, since the mutated PVI's pole structure differs enough that
the wedge selection can't reach the winding-ring targets.  All 15
downstream assertions error.

M3 bites identically to Fig 2's M3 — refuse mode + winding ring at
`Re < 0` corners the walker.

M4 bites in a textbook clean way — 5 distinct assertions fail across
3 testsets, all stemming from the sheet-counter staying at `[0]`
throughout: sheet [1] queries return NaN (no matching visited node),
the `maximum(empty)` in FF3.1.5 errors, and the FF3.1.6 phase
distinction collapses.

The plan's M2 ("replace `angle(w)` with `abs(w)` in render") is a
figure-only mutation — the test file evaluates `angle(w)` directly
via `eval_at_sheet` and doesn't go through the figure render path,
so M2 would have ZERO test bite.  Per the honest-deviation latitude:
not applied, documented here.  The figure-script's HSV phase-
portrait rendering is acceptance-tested **visually** against FFW
Fig 3 — see `figures/output/ffw2017_fig_3.png`.

## What is NOT shipped

  - **Full FFW Fig 3 z-window** — FFW's `|z| ∈ [1/100, 10]` ↔
    `Re ζ ∈ [-4.6, 2.3]`; we ship `[-1.5, 2.5]`.  Extending toward
    `Re ζ = -4.6` runs into the same pole-density growth near ζ = 0
    that FFW md:178 flags ("there can be high pole densities in the
    neighborhoods of the branch points") and would require a denser
    target grid in the low-Re ζ region — Poisson-disk node placement,
    same as bead `padetaylor-zwh`.

  - **FFW's two-sheet z-plane geometry** — FFW Fig 3's caption
    distinguishes sheet 0 (cut on `z ∈ (0, 1)` only) from sheet 1
    (cut additionally on the negative real z-axis).  Our v1 z-plane
    sheet-1 panel renders `arg u(z)` over the full Cartesian
    `|z| ≤ exp(Re ζ_HI)` disc using `ζ = log|z| + i(arg z + 2π·1)`
    bilinear-interp from the sheet-[1] walker data; the negative-
    real cut is visible in the rendered output but is not explicitly
    annotated.  Adding cut-annotation overlay is a Makie-side
    cosmetic upgrade.

  - **rng_seed independence** — the walker uses `rng_seed = 2` to
    reproducibly populate sheet [1].  Under `rng_seed = 0` (the
    default) the walker NEVER crosses the cut for our target set;
    the figure is bound to this specific seed.  Documented in the
    figure-script docstring; the principled fix is Poisson-disk
    target placement (deferred).

  - **M2 phase-vs-modulus mutation** — would not bite the test file
    (which evaluates `angle(w)` via `eval_at_sheet` directly, not
    through the figure render path).  Not applied; documented above.
