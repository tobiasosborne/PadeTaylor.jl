## Design synthesis

The PVI fixed branch points are z = 0 and z = 1.  Under `z = e^ζ`,
z = 0 maps to ζ = -∞ (out of the finite plane); z = 1 maps to the
periodic lattice `ζ = 2π·i·k` for k ∈ ℤ.  Under `ζ = e^η`, the
ζ = 0 lattice point (k = 0) maps to η = -∞; remaining `k ≥ 1` points
sit at `η = log|2π·k| + i·arg(2π·i·k)`, outside the branch-point-
free region `Re η < log(2π) ≈ 1.838`.

This makes PVI the natural test for the full multi-method stack:

  | Column | Frame | Method                | Branches in play          |
  |--------|-------|-----------------------|---------------------------|
  | 1      | η     | A3 (no constraints)   | none in finite η-region   |
  | 2      | ζ     | A4 refuse             | ζ = 0 (sheet-0 only)      |
  | 3      | ζ     | A4 cross + A5         | ζ = 0 (sheets [0] + [1])  |

All three render the same analytic solution — same `|u|` field
under the respective parameterization.  The FFW figure's two-row
layout (showing each column's two sheets) is here flattened to a
3-column × 2-row layout for v1 simplicity (column 3 is the only
one with two rows; the others use the single-strip equivalent).

### Why a "winding ring" of targets in column 3

Cross-mode allows the walker to cross cuts, but it doesn't force the
walker to do so.  With min-|u| wedge selection on a rectangular
target grid in the upper-half-plane, the walker naturally walks UP
without circumambulating ζ = 0 — visited_sheet stays at `[0]`
throughout, and sheet [1] queries return NaN.

To populate sheet [1] reproducibly, the column-3 target list adds 8
"winding ring" targets at `(-0.7, ±0.5), (-0.5, ±0.7), (-0.3, ±0.9),
(-0.1, ±1.1)`.  These force the walker to take paths that wind
around ζ = 0, picking up sheet bumps at each cut crossing.  This is
documented in the figure script's header docstring as the explicit
choice (CLAUDE.md Rule 9); the principled alternative (Poisson-disk
sampling that naturally circumambulates) is deferred.

## What is NOT shipped

  - **Two-row FFW layout faithfulness** — FFW's Fig 2 shows each
    column with two rows (η-plane: sheets 0 and 1; ζ-plane: sheets
    0 and 0+1 reached differently).  Our v1 flattens column 1 and 2
    to single panels (combined sheet view in col 1; single sheet in
    col 2); only column 3 has two stacked panels.  The 6-panel layout
    can be added incrementally without algorithmic changes.

  - **FFW's exact tolerance pin** — FFW md:195 reports symmetry-
    based estimates 4e-7 / 3e-6 (η), 6e-7 / 6e-4 / 5e-4 (ζ).  Our
    loose-vs-tight pins are one order looser (4e-6 / 6e-6 / ...)
    per the worklog 036 convention.

  - **PainleveProblem forwarding** — figure script builds
    PadeTaylorProblem directly (bypassing PainleveProblem) since
    Painleve.jl doesn't yet forward `branch_points` /
    `branch_cut_angles` / `cross_branch` / `grid_sheet` kwargs.
    Trivial follow-up; not in this bead.
