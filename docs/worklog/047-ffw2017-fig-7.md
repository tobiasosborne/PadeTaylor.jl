## Multi-branch-point investigation (NEW risk handled inline)

The Fig 7 plan flagged that PVI's second fixed branch point (`z = 1`
↔ `ζ = 2π i k`) lives inside our rendering window (`Im ζ ∈ (-π, 3π]`
straddles `Im ζ = 2π`), and that prior Figs 2/3 used
`branch_points = (0im,)` because the tronquée's pole-free sector
kept the walker away from `Im ζ ≈ 2π`.  For generic PVI we did not
know whether the walker would error near `ζ = 2πi`.

**Action taken**: read `src/BranchTracker.jl` lines 71-81 first.  The
kwarg contract:

> `branch_points` is a tuple of `Complex` (any element type
> compatible with the problem's `T`).  Empty tuple is the no-op.
> `branch_cut_angles` is a scalar `Real` (broadcast to all
> branches) or a tuple of the same length as `branch_points`.

Confirmed multi-point tuples are supported with no further wiring.
Fig 7 ships with:

```julia
BRANCHES   = (0.0+0.0im, 0.0 + 2π*im, 0.0 - 2π*im)
CUT_ANGLES = (Float64(π), Float64(π), Float64(π))
```

(three branch points, all cuts along `arg = π`).  `visited_sheet`
now tracks a `Vector{Int}` of length 3 per visited node, one counter
per branch.  Empirical: walker survives across the `Im ζ = 2π`
neighbourhood with no errors; the per-branch counters resolve to
`[0, 0, 0]` for the principal sheet and `[1, 0, 0]` for the
alternate sheet reached via the `ζ = 0` cut crossing.  The
`ζ = ±2πi` counters remain `0` throughout the walked path under our
rng_seed=1 + winding-ring geometry (the walker never circumambulates
those secondary branches).

**Decision retained**: keep all three branch points in the tuple
even though only the first counter is non-zero.  Defensive: the
counters are cheap (3 Int per node), and shipping the architecture
ready for cases where higher-Im ζ targets WOULD force circumambulation
of `ζ = 2πi` is preferable to a v1 that hard-codes `branch_points =
(0im,)` and would need re-tuning later.

## Mutation M2 honest non-bite

The plan predicted M2 (drop `node_separation = R_ζ`) would bite via
"walker bogs/errors on generic pole field → ERROR cascade".
**Empirical: it does NOT bite.** At our window + target density:

  - With `node_separation = R_ζ`: 205 visited nodes; visited_h spread
    0.024–0.390 (R(ζ)-driven, well-spread).
  - Without: 1536 visited nodes; visited_h spread 0.008–0.331
    (`:adaptive_ffw` shrinks h adaptively, also well-spread; the
    smaller initial `h_cur` widens the walker's reach so it
    visits MORE nodes, not fewer).

The two policies are complementary (both produce h-variation) but
neither is necessary for correctness at this window.  `node_separation`
is a **figure-side ergonomic optimisation** — it gives a more
compact visited tree with comparable rendering quality at lower
wall-time (1.99 s vs ≈ 8 s).  Marked as v1 corner per CLAUDE.md
Rule 9.

For completeness we added **FF7.1.5b** as a sanity pin (visited_h
spread > 0.05) which both modes satisfy; it catches a "fixed h
everywhere" regression but does NOT discriminate M2.

## What is NOT shipped (CLAUDE.md Rule 9)

  - **FFW's three-strip per-strip pins (5e-8 / 4e-4 / 8e-4, md:301).**
    Our v1 renders a single principal-sheet η-strip; the FFW figure
    shows three strips with per-strip Schwarz-reflection error
    estimates.  Our FF7.1.5 measures `max ≈ 2.8e-4` over 24 sample
    points (median 4.9e-12), comparable to FFW's middle-strip `4e-4`
    pin — but the load-bearing acceptance is the captured tight-tol
    absolute pins (FF7.1.3, FF7.1.4), not the per-strip error.
    Extending to three strips would need the same Poisson-disk
    target-placement infrastructure as bead `padetaylor-zwh` (FFW
    Fig 1's wider window).

  - **Full FFW Fig 7 z-window.**  FFW shows `|z| ∈ [some range]`
    (specific bound not stated in caption; visually appears similar
    to Fig 6's `|z| ∈ [0.37, 12.2]`).  We ship the central annulus
    `|z| ∈ [0.45, 4.48]` (`Re ζ ∈ [-0.8, 1.5]`) to keep the figure-
    script wall ≤ 20 s.  Same Poisson-disk-node lift would extend
    this.

  - **rng_seed-independent reproduction.**  Walker uses `rng_seed = 1`
    to reproducibly populate the alternate sheet; under default
    seed=0 alt-sheet population drops to 11 nodes (still > 5 the
    test threshold but visibly sparser in the figure).  Documented
    in the figure-script docstring; the principled fix is Poisson-
    disk target placement (deferred).

  - **`ζ = ±2πi` circumambulation.**  Architecture is ready (3
    branch points, 3 counters), but at our window the walker never
    circumambulates the secondary branches.  Higher-Im ζ targets
    would exercise this; deferred.

  - **Mutation M2 (drop node_separation) bite.**  Discussed in
    §"Mutation M2 honest non-bite" above.  The kwarg is figure-side
    ergonomic; no test surface discriminates it.
