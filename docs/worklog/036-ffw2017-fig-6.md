## Ground truth read first

Per Law 1, first reading was the FFW 2017 preprint sections relevant
to Fig 6:

    rescale, unlike PIII) and the modified equation `P̃_V`:

        w'' = (1/(2w) + 1/(w-1)) (w')²
              + (w - 1)² (α w + β/w)
              + γ e^ζ w
              + δ e^(2ζ) w (w + 1) / (w - 1)

    (poles/zeros for the generic PV solution). Quoted FFW md:297:

## The reduced ζ-window

FFW md:281-297 do not explicitly state the ζ-window dimensions; the
figure (`_page_16_Figure_2.jpeg`) shows three contiguous strips of
vertical width `2π` (PV strip width per md:103) stacked above each
other, with `Re ζ` running roughly from a small negative value to a
few positive values. We adopt:

    Re ζ ∈ [-1, 2.5]
    Im ζ ∈ (-π + 0.01, 5π - 0.01]
    sheet 0: Im ζ ∈ (-π,  π],   z-arg ∈ (-π,  π]
    sheet 1: Im ζ ∈ ( π, 3π],   z-arg ∈ ( π, 3π]
    sheet 2: Im ζ ∈ (3π, 5π],   z-arg ∈ (3π, 5π]

The `Re ζ ∈ [-1, 2.5]` range corresponds to `|z| ∈ [e^{-1}, e^{2.5}]
≈ [0.37, 12.2]` in the z-plane — a finite annulus around `z = 0`,
visited three times as one winds around the fixed branch point.

The strip-boundary epsilon (`±0.01`) keeps Stage-2 evaluations inside
the visited tree's reach: a sample exactly at `Im ζ = 5π` could fall
just outside the nearest visited node's `h`-disc, producing a `NaN`
under the fail-loud Stage-2 contract (`PathNetworkSolution` returns
`NaN + NaN·im` when the nearest visited node is farther than its
`visited_h` — see `src/PathNetwork.jl:493-507`).

## The node-separation function R(ζ)

FFW Fig 1 prescribes `R(ζ) = (8 - Re ζ)/20` for PIII over `Re ζ ∈
[-2, 8]`. We adapt to our reduced `Re ζ ∈ [-1, 2.5]` patch with the
same FFW shape (linear decrease in `Re ζ`, scaled appropriately):

    R(ζ) = max(0.05, (4 - Re ζ) / 10)

At `Re ζ = -1` it gives `R = 0.5` (FW 2011's working `h` default;
matches the IC step); at `Re ζ = 2.5` it gives `R = 0.15` (~3×
denser steps in the higher-density region). The `0.05` floor
prevents zero-spacing at the high boundary — without it, R reaches
`R(4) = 0` and the walker stalls.

This is a **scaled-and-floored variant** of FFW's prescription, not
verbatim: the original was tuned for the full FFW PIII Fig 1 domain,
not our reduced PV patch. The scaling factor `1/10` instead of `1/20`
sets the typical step in the high-density region to ~`R ≈ 0.15`,
roughly matching what a `(8 - 6)/20 = 0.1` would have given at the
top of FFW's PIII patch.

## What shipped

### `figures/ffw2017_fig_6.jl` (new, ~225 LOC including the
~70 LOC top-of-file docstring)

Builds the `P̃_V` problem via `pV_transformed_rhs(1, -1, 1, -1/2)`
factored over the FFW md:47 RHS. Maps the IC `(z₀, u₀, u'₀) = (1, 2,
-1)` to the ζ-frame: `(ζ₀, w₀, w'₀) = (0, 2, -1)` (since `ζ = log z`,
`w = u`, `w' = z u'` and `z₀ = 1`). Walks via
`path_network_solve` with `step_size_policy = :adaptive_ffw`,
`adaptive_tol = 1e-10`, `k_conservative = 1e-3`, and `node_separation
= R`. Stage-1 targets are placed at `R(ζ)` density (~867 targets);
Stage-2 evaluation lattice is a uniform `90 × 240` rectangle (~21,600
cells), with **separate** stage-1 vs stage-2 grids so the figure can
render at a finer resolution than the walker walks (the Stage-2
no-cover boundary turns into `NaN` at ~2% of cells, plotted as
missing).

Renders nine panels:
  * **Column 1**: `|w(ζ)|` heatmaps per sheet — modulus surface in
    the transformed plane, three vertical strips.
  * **Column 2**: `|u(z)|` z-plane scatter-heatmaps per sheet — points
    placed at `(r cos θ, r sin θ)` with `r = exp(Re ζ)`, `θ = Im ζ`.
    The "spirals around z = 0" FFW md:283 calls out are immediately
    visible.
  * **Column 3**: `arg u(z)` phase portrait per sheet — HSV-encoded
    `arg ∈ [-π, π]`, rotating around each pole/zero.

PNG: `figures/output/ffw2017_fig_6.png`. Visible features: pole
spikes in column 1 form irregular vertical clusters within each
strip; column 2 spirals are most prominent in sheet 0 (smallest
radii); column 3 phase portraits show complex multi-color rotation
patterns at each pole and zero — consistent with FFW md:283's
"oblique lines" signature.

### Mutation-proof bites (verified, footer-documented)

  - **M1**: Swap `α ↔ β` to `(-1, 1, 1, -1/2)`. The PV problem changes
    character; sheet-0 `u_tight - u_ref` pin error becomes `0.40`
    (threshold `1e-10`) — bites **FF6.1.3 absolute pin**. Sheet-2 pin
    error becomes `1.80` — bites **FF6.1.5 absolute pin**. The
    loose-vs-tight convergence pin alone does NOT bite (mutation
    preserves controller self-consistency); the absolute pins are
    the load-bearing bite-detectors here.

  - **M2**: Drop `step_size_policy = :adaptive_ffw`, fall back to
    `:fixed` with `h = 0.5` (and necessarily drop `node_separation`
    since R + :fixed yields a different regime). The walker takes
    too-coarse steps through the high-Im-ζ pole-dense region.
    Sheet-2 `u_tight - u_ref` pin error becomes `0.60` — bites
    **FF6.1.5 absolute pin**. Loose-vs-tight convergence: :fixed
    ignores tol, so identical → does NOT bite.

  - **M3**: Drop `node_separation` only (keep `:adaptive_ffw`). The
    walker **THROWS** at the first high-Re-ζ target: `target
    unreachable in 4000 steps`. Adaptive alone has no spatial-density
    signal: it shrinks `h` only when `T(h) > Tol` (a temporal
    truncation signal), but the FW-2011 default `h = 0.5` is already
    too large to thread the high-Re-ζ pole density before the
    `max_steps_per_target` cap. ADR-0012 alternative D rejected for
    exactly this reason; M3 is the empirical confirmation. The
    strongest possible bite — the test setup itself fails before any
    assertion runs.

All three mutations bite the test, each at a different assertion
class. Test catches them.

## Frictions

### One thing surprised me — the strip-boundary epsilon

The first probe set `Im ζ ∈ [-π, 5π]` (closed interval at both ends).
Stage-2 evaluations at `Im ζ = -π` or `5π` came back `NaN` because
the nearest visited tree node was just slightly outside the boundary
(walker can land within `R(z)` of a target but not exactly on it,
and `R ≈ 0.05-0.5` near the boundary). The `±0.01` shrink moves the
boundary cells safely inside the visited-tree's reach.

This isn't a bug in `path_network_solve`; it's a feature of FFW
md:93's "walker terminates within `|qh|` of the target" contract.
Worth a docstring sentence in the figure script for future readers.

### The Stage-1 / Stage-2 grid split

Initial probe used a single combined grid (Stage-1 targets =
Stage-2 evaluation cells), which gave 87% coverage on the eval
lattice — the cells where adaptive shrunk `visited_h` below `R(z)`
ended up outside their nearest visited node's `h`-disc. Splitting
into a coarser Stage-1 target set (at `R(ζ)` density) and a finer
uniform Stage-2 evaluation lattice raised coverage to 97.7%. This
is the FW 2011 §3.1 design (`md:155-166`) — Stage 1 builds the path
tree, Stage 2 reads it at whatever resolution you want — and the
figure script's `figures/fw2011_fig_3_1.jl` already does it the
same way for FW Fig 3.1. Should have led with that idiom; mild
self-friction at first.

### M1 absolute-value pins, the convergence pin's gap

The original test design had only the loose-vs-tight tolerance
convergence pin (FFW Table-2-style self-cross-check). It DOES bite
M2 (sheet-2 error blows up under :fixed). It does NOT bite M1 (α ↔ β
swap), because the mutation preserves the walker's self-consistency
— both `u_loose` and `u_tight` move together to the new physical
solution, and their difference stays small. The absolute-value pins
against the baseline-captured `(1.953, -0.529)` for sheet 0 and
`(1.217, -0.466)` for sheet 2 are what bite M1.

This is a useful diagnostic. The convergence pin catches
**numerical-discipline** mutations; the absolute pin catches
**physical-problem** mutations. A full test needs both for
mutation-completeness.

## Hard-won lesson

**Adaptive `h` ≠ non-uniform nodes.** They compose; one is not a
substitute for the other. M3 walked into this immediately: with
`:adaptive_ffw` alone and a fixed `h = 0.5` initial step, the FW
2011 defaults survive the smooth low-Re-ζ region (where pole
spacing > 0.5) but choke the moment a per-target walk needs to
thread a high-Re-ζ pole-dense region (where pole spacing << 0.5).
The controller shrinks `h` via `q ≤ 1` only on a truncation-error
signal, which is the *temporal* error; the *spatial* density is
information the controller never sees.

`node_separation = R` provides that spatial-density signal
out-of-band: the walker's step magnitude is bounded above by
`R(z_cur)`, so a high-density region prescribes a small step
regardless of what the local truncation error happens to be. The two
mechanisms close orthogonal gaps. M3 demonstrates this empirically;
ADR-0012 alternative D (skip R, rely on adaptive alone) is rejected
not by argument but by failure.

The general rule for the rest of the FFW figure pipeline (Figs 1, 2,
3, 4, 5, 7 to come): **always pass both** `step_size_policy =
:adaptive_ffw` AND `node_separation = R`. The FFW workflow is "node
density tracks pole density, controller refines below that ceiling"
— one without the other is a partial implementation that breaks at
exactly the regime FFW was published to solve.

Quantitative pin summary, from the test run:

  | sheet | FFW (md:297) | this work | threshold (1 order looser) |

We beat FFW's numbers by 2-3 orders on each sheet — almost certainly

their PIII Fig 1 PoC.

`90 × 240` Stage-2 lattice, single-threaded order=30 Float64.

## What is NOT shipped

  - **Full FFW window**. We render `Re ζ ∈ [-1, 2.5]`; FFW's published
    figure likely covers a wider domain. The acceptance criterion
    (one order of magnitude on per-sheet error) is met; widening is
    an aesthetic improvement, not a correctness one. File a bead if
    a future reader needs the visual fidelity for a publication
    figure.

  - **FFW Table-2-style symmetry-method error estimate**. FFW md:122
    estimates error as `|w(ζ) - w(\bar ζ)| / |w(ζ)|` exploiting the
    upper-lower-half conjugate symmetry. The test uses the
    loose-vs-tight tolerance method instead, which is principled
    (a tighter tol is a better reference) but different. A bead to
    add the symmetry-method estimator as a complementary error
    measure is worth filing if a downstream consumer wants
    FFW-comparable numbers verbatim.

  - **Figures 1, 4, 5 from FFW 2017 PIII/PV tronquée**. These are
    steps B2, B3, B4 of the 11-step plan; out of scope for B1.
